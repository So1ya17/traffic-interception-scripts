# mitmproxy — Перехват и анализ трафика

Набор скриптов для перехвата сетевого трафика через mitmproxy с подробным логированием ошибок и генерацией HTML-отчётов с графиками.

## Структура проекта

```
mitmproxy/
├── capture.py          # Запуск mitmproxy с записью трафика
├── _capture_addon.py   # Addon для подробного логирования (HTTP/WS/TCP)
├── export.py           # Генерация HTML-отчёта из .flows файла
├── sslkeys.log         # SSL ключи для Wireshark (авто-генерируется)
├── report.html         # Пример отчёта
└── capture_*.flows     # Файлы с захваченным трафиком
```

## Требования

```bash
pip install mitmproxy
```

## Быстрый старт

### 1. Запуск захвата трафика

```bash
# Базовый запуск (порт 8080, файл с временной меткой)
python capture.py

# Указать имя файла
python capture.py -o mylog

# Другой порт
python capture.py -p 9090
```

### 2. Настройка прокси на устройстве

**iPhone/iPad (iOS):**
1. Настройки → Wi-Fi → нажать `i` на сети → Прокси → Вручную
2. Сервер: `<IP вашего ПК>` (узнать через `ipconfig`)
3. Порт: `8080`
4. Открыть `http://mitm.it` в Safari и установить сертификат

**Android:**
1. Настройки → Wi-Fi → удерживать сеть → Изменить сеть → Прокси → Вручную
2. Имя хоста: `<IP вашего ПК>`
3. Порт: `8080`

**Windows (системный прокси):**
1. Win + I → Сеть и Интернет → Прокси → Настройка прокси вручную
2. Адрес: `127.0.0.1`, Порт: `8080`

### 3. Экспорт в HTML

```bash
python export.py capture_20260629_112944.flows

# Указать имя выходного файла
python export.py capture_*.flows -o report.html
```

## Опции capture.py

| Флаг | Описание | По умолчанию |
|------|----------|--------------|
| `-o, --output` | Имя выходного файла (без расширения) | `capture_<дата>.flows` |
| `-p, --port` | Порт прокси | `8080` |
| `--rotate N` | Ротация файлов каждые N минут | Без ротации |
| `--rotate-size N` | Ротация каждые N мегабайт | Без ротации |
| `--no-sslkey` | Отключить запись SSL ключей | Запись включена |

## SSL расшифровка в Wireshark

При запуске `capture.py` автоматически создаётся файл `sslkeys.log` с TLS ключами.

**Настройка Wireshark:**
1. Edit → Preferences → Protocols → TLS
2. В поле `(Pre)-Master-Secret log filename` указать путь к `sslkeys.log`
3. Теперь HTTPS трафик расшифровывается

## Что логирует addon

Все запросы выводятся в реальном времени с цветовой кодировкой:

| Цвет | Тип | Описание |
|------|-----|----------|
| Зелёный | `OK` | Успешный ответ (2xx) |
| Жёлтый | `REDR` | Редирект (3xx) |
| Красный | `ERR4` | Ошибка клиента (4xx) |
| Красный жирный | `ERR5` | Ошибка сервера (5xx) |
| Красный | `FAIL` | Connection error / timeout |
| Фиолетовый | `WS` | WebSocket сообщения |
| Синий | `TCP` | TCP соединения |

## HTML-отчёт

Отчёт включает:

### Графики
- **Время ответа** — горизонтальные бары + P50/P95/P99
- **Распределение статусов** — donut-диаграмма (2xx/3xx/4xx/5xx)
- **Топ-10 хостов** — бары со средним временем ответа
- **Таймлайн** — scatter plot (точка = запрос, высота = время ответа)
- **HTTP методы** — цветные пилюли (GET/POST/PUT/DELETE)

### Фильтрация
- Текстовый фильтр по URL, хосту, методу
- Кнопки: Ошибки / Медленные / Таймауты / Refused / 5xx

### Детали запросов
- При клике на карточку раскрываются заголовки и тело запроса/ответа
- Красная полоса для ошибок, жёлтая для медленных
- Бейджи: TIMEOUT, REFUSED, 5xx, SSL, DNS

## Примеры использования

### Перехват трафика приложения на iPhone

```bash
# Запуск на ПК
python capture.py -o iphone_traffic

# На iPhone: настроить прокси на <IP_ПК>:8080
# Открыть http://mitm.it и установить сертификат
# Запустить приложение
# Остановить capture (Ctrl+C)

# Экспорт отчёта
python export.py iphone_traffic.flows -o report.html
start report.html
```

### Долгая сессия с ротацией

```bash
# Ротация каждые 10 минут
python capture.py --rotate 10 -o long_session

# Ротация каждые 50 MB
python capture.py --rotate-size 50 -o large_traffic
```

### Объединение нескольких файлов

```bash
python export.py capture_*.flows -o combined_report.html
```

## Диагностика проблем

### Белый экран при запуске приложения

Если приложение показывает белый экран, проверьте:
1. **Sentry 400 ошибки** — невалидный DSN ключ
2. **ATT rate limiting** — множественные вызовы `trackingAuthorizationStatus`
3. **Background task assertions** — приложение создаёт/уничтожает assertions в цикле
4. **Sandbox denies** — попытка обращения к запрещённым ресурсам

### Медленная загрузка

В HTML-отчёте нажмите **Медленные** для фильтрации. Ищите:
- `connection_reused` с большой задержкой
- DNS резолв дольше 100ms
- TLS handshake дольше 500ms

### Приложение не подключается к серверу

В логе capture.py ищите:
- `CONNECTION TIMEOUT` — сервер не отвечает
- `CONNECTION REFUSED` — порт закрыт
- `CONNECTION RESET` — сервер разорвал соединение
- `SSL/TLS ERROR` — проблема с сертификатом
- `DNS ERROR` — не удаётся разрешить имя
